# LC-100 Subsets Recursive

**Category**: Recursion
**Difficulty**: Medium

## Algorithm

Given a list of N distinct non-negative integers, enumerate all 2^N subsets
(the power set) including the empty set and the full set. Output one subset per
line; within each subset print the elements in the original input order,
space-separated; print nothing (just a newline) for the empty subset. Output
subsets in the order they are generated by a recursive include/exclude traversal.

The recursive construction at each step processes one element: either include it
in the current subset or exclude it, then recurse on the remaining elements. The
two recursive branches produce all subsets containing the current element and all
subsets not containing it, respectively.

Include-first traversal: for element at index `i`, first recurse with the
element added to the working set (index `i+1`, size `k+1`), then recurse
without it (index `i+1`, same size `k`). This yields a specific but consistent
ordering across all subsets.

For N distinct elements the total output is exactly 2^N subsets regardless of
their values.

## Input format

```
Line 1: N
Line 2: N space-separated distinct non-negative integers
```

## Oracle

- N=2, elements=[1,2] → four lines: `1 2`, `1`, `2`, `` (empty)
- N=3, elements=[1,2,3] → eight lines in include-first order

## Approach hint

Recursive helper `fn gen(elements, n, depth, buf, buf_len)`. Base case:
`depth == n` — print the buf contents space-separated then newline. Recursive
case: set `buf[buf_len] = elements[depth]`, recurse with `buf_len + 1`
(include); then recurse with same `buf_len` (exclude).

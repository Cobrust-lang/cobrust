# Module: pkg

## Purpose

Cobrust package format — the M12 deliverable. Provides:

- A canonical user-crate `cobrust.toml` parser + validator.
- A deterministic `cobrust.lock` writer + reader.
- A max-compatible greedy semver resolver.
- A content-addressed registry rooted at `~/.cobrust/registry/blake3/<hex>/`.
- Path / git / registry source backends.
- Deterministic-tarball helper for content-addressing source trees.

Constitution `CLAUDE.md` §2.2 binds the deliverable:

> `__init__.py` / sys.path / packaging chaos → **single canonical
> package format, content-addressed, one tool**

§2.4 binds the determinism contract:

> **Deterministic build IDs**: hash of source + toolchain + LLM router
> decisions, so any translation is reproducible bit-for-bit given
> the same inputs.

## Status

- **M12 — delivered.** ADR-0026 binds the schemas + algorithms.
  Closes ADR-0019 §"M12" + line 2 of §"Definition of usable for most
  projects".

## Manifest schema (ADR-0026 §"Manifest schema (binding)")

```toml
[package]
name = "my_app"                          # required; [a-zA-Z][a-zA-Z0-9_-]*
version = "0.1.0"                        # required; semver
cobrust-version = "0.0.1"                # required; M12 pins to 0.0.1
authors = ["Alice <alice@example.com>"]  # optional
license = "Apache-2.0 OR MIT"            # optional
description = "Short description"        # optional

[dependencies]
cobrust-tomli = { path = "../cobrust-tomli" }                           # path
my_lib       = { git = "https://...", rev = "abc123" }                  # git
serde-like   = "1.2"                                                    # registry

[dev-dependencies]
test_helpers = { path = "./test_helpers" }

[bin]
name = "my_app"
path = "src/main.cb"

[lib]
name = "my_app_lib"
path = "src/lib.cb"

[[test]]
name = "smoke"
path = "tests/smoke.cb"
```

### Validation rules

| Field | Rule |
|---|---|
| `package.name` | `^[a-zA-Z][a-zA-Z0-9_-]*$`; ≤ 64 chars |
| `package.version` | Valid semver per `semver::Version::parse` |
| `package.cobrust-version` | Valid semver; M12 pins `0.0.1` |
| `dependencies.<key>` name | Same as `package.name` |
| `dependencies.<value>` | One of: bare-string semver, `{ path = ... }`, `{ git = ..., rev = ... }`, `{ version = ..., registry = ... }` |
| Mutual exclusion | `path` source rejects sibling `version` / `git` / `registry`; `git` requires `rev` |
| At least one of `[bin]` / `[lib]` | required |
| `[bin].path == [lib].path` | rejected (`ConflictingPaths`) |
| `[router]` present + `[package]` absent | rejected (`IsRouterConfig`) |

## Lockfile schema (ADR-0026 §"Lockfile schema (binding)")

```toml
# cobrust.lock — auto-generated by `cobrust build`. Do not edit.
# Determinism contract: ADR-0026 §C.

[metadata]
manifest_hash = "blake3:<hex>"
lockfile_version = 1

[[package]]
name = "cobrust-tomli"
version = "2.0.1"
source = "path+file:///abs/path/to/cobrust-tomli"
hash = "blake3:39e9e2d3..."
provenance_hash = "blake3:39e9e2d3..."
dependencies = ["serde-like 1.2.3"]

[[package]]
name = "my_app"
version = "0.1.0"
source = "path+file:///abs/path/to/my_app"
hash = "blake3:root"
dependencies = ["cobrust-tomli 2.0.1"]
```

### Determinism contract

- `[[package]]` entries sorted by `(name, version)` ASCII-lex.
- `dependencies = [...]` sorted lex.
- Field order within `[[package]]`: `name, version, source, hash,
  provenance_hash, dependencies`.
- LF-only line endings; trailing newline.
- BTreeMap-backed everything; canonical TOML serializer hand-written.
- `manifest_hash` = `blake3(Manifest::canonical_toml())` — independent
  of user's table-key ordering in the on-disk manifest.

## Resolution algorithm (ADR-0026 §D)

```text
fn resolve(manifest, registry) -> Lockfile:
    graph = DepGraph::new()
    walk(manifest.package, manifest.dependencies, graph)
    chosen = max_compatible(graph)
    if has_cycle(graph): err Cycle
    Lockfile::from_resolution(manifest, chosen)
```

Failure modes:

| Error | Trigger |
|---|---|
| `Conflict { package, reqs }` | No version satisfies all sibling reqs |
| `Cycle { path }` | SCC in dep graph (DFS-with-color) |
| `MissingPackage { name }` | Registry has no entry for the requested name |
| `Offline` | Registry source consulted at M12 (stub) |
| `PathMissing` | Path source's directory doesn't exist |

`ResolutionStrategy` is trait-shaped so a future Phase F PubGrub-backed
solver replaces the strategy without breaking the public surface.

## Registry layout (ADR-0026 §F)

```text
~/.cobrust/registry/
├── blake3/
│   ├── 39e9e2d3...0069/      # cobrust-tomli 2.0.1
│   │   ├── cobrust.toml
│   │   ├── PROVENANCE.toml   # ADR-0007 chain-of-custody
│   │   └── src/...
│   └── deadbeef.../          # ...
└── index/
    └── name-to-versions.toml  # cached index (M12 stub; empty)
```

`Registry::insert_source_tree` recipe:

1. `Tarball::build(source_dir)` — sorted entries, zeroed mtime/uid/gid,
   canonical perms 0o644 / 0o755, gzip level 6.
2. `blake3(gzipped tarball bytes)` keys the cache directory.
3. Atomic extract: stage in `<root>/blake3/.staging-<hex>`, rename on
   success.
4. Idempotent: if `<root>/blake3/<hex>/` already exists, no-op.

## Tarball determinism (ADR-0026 §G)

Excluded from tarballs (always):

- `target/` — build outputs
- `cobrust.lock` — build outputs
- `.git/` — VCS metadata
- `.cobrust/` — agent-local state
- symlinks (unconditionally dropped at M12)

Same source tree → identical bytes → identical blake3 hex.

## Source resolvers (ADR-0026 §E)

| Source | Behavior |
|---|---|
| `Path { path }` | Resolve relative to manifest dir; insert into registry |
| `Git { url, rev }` | Invoke system `git clone --depth=1 --branch=rev`; fall back to plain clone + `checkout rev`; insert into registry |
| `Registry { name, version }` | M12 stub: returns `RegistryError::Offline` unless cached |

## Provenance integration (ADR-0026 §H)

Translated crates carry `PROVENANCE.toml` with a `deterministic_id =
"blake3:<hex>"` field. The pkg pipeline reads this at fetch time
(via `sources::read_provenance_hash`) and stores it in the lockfile's
`provenance_hash` column. ADR-0007 chain-of-custody round-trips
end-to-end.

## Non-goals (M12)

- HTTP-fetch from a remote registry. M12 returns `Offline` for any
  registry-source dep not already in the local cache; Phase F lands
  the full registry under a separate ADR.
- Multi-`[bin]` / multi-`[lib]` crates. M12 ships single `[bin]` +
  single `[lib]` per ADR-0026 §A.1; future Phase F may lift to
  `[[bin]]` if multi-bin crates materialize.
- `import` statement → multi-source-file linking. The M12 build
  compiles the single `[bin].path` source file. Multi-source-file
  orchestration is M12.x scope.
- Aggregate / Ref / Cast Rvalue codegen materialization. These are
  M11 followups (ADR-0025 §F) tracked as deferred-past-M12 per the
  prompt's "smallest correct increment" rule. The notebook example
  works around this by using literal-string `print(...)` callsites
  end-to-end (the tier the M11 codegen lowers).

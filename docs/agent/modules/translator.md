# Module: translator

## Purpose

L0 → L3 closed-loop translation of Python (and later C / C++ /
Fortran) libraries into Cobrust, with full provenance.

## Status

- **M4 — delivered.** End-to-end pipeline runs against `corpus/tomli`
  in synthetic-LLM mode. 31 unit tests + 5 integration tests = **36
  tests pass**. The downstream `cobrust-tomli` crate is generated by
  the pipeline and gated by L0..L3 evidence (see `mod:tomli`).

## Pipeline

```
target source
      ↓
   L0 spec extraction
      ↓
   L1 translation
      ↓
   L2 verification ──┐
      ↓              │ retry (diagnostic-driven)
   L3 integration    │
      ↓              │
   Cobrust registry  │
                     │
   gate failure ─────┘
```

## Modes (M4 — pinned by `adr:0007`)

| Mode | Provider | Default? | Cargo feature |
|---|---|---|---|
| Synthetic | `crate::synthetic::SyntheticProvider` from canned TOML | yes | none |
| Real-LLM | `cobrust_llm_router::{AnthropicProvider, OpenAiProvider}` | no | `real-llm` (M5+) |

The synthetic-LLM mode serves pre-recorded responses keyed by
`(task, function)` — see [`adr:0007`](../adr/0007-translator-pipeline.md)
§"Synthetic-LLM mode" for the contract. The translator embeds a
header on every prompt so the synthetic provider can route without
parsing the body:

```text
cobrust-translator/v1
task: <task>
function: <function>
source-sha256: <16-hex>
---
<prompt body>
```

## Error taxonomy

`TranslatorError` variants:

| Variant | Meaning |
|---|---|
| `SpecExtraction(String)` | L0 failed (spec.toml not found / malformed) |
| `Translation { function, message }` | L1 failed for one function |
| `BuildGate(String)` | L2.build failed |
| `BehaviorGate(String)` | L2.behavior failed |
| `DownstreamGate(String)` | L3 failed |
| `SyntheticMiss { task, function }` | synthetic provider had no canned response (permanent; caller must add or switch to real-LLM); surfaces internally as `LlmError::Provider { code: "synthetic-miss" }` and is also raised as `synthetic-stale` when `source_sha16` mismatches |
| `Io(io::Error)` | filesystem failure |
| `Router(RouterError)` | router-level failure |
| `Manifest(String)` | manifest validation failure |
| `Config(String)` | configuration parse failure |
| `Decode(String)` | TOML / JSON decode failure |

## Gates (none optional)

| Stage | Gate | Pass criteria | M4 |
|---|---|---|---|
| L0 | spec produced | `spec.toml` + `harness/` committed | tomli ✅ |
| L1 | code emitted | every file has provenance header | tomli ✅ |
| L2.build | `cargo build --release` | zero warnings | tomli ✅ |
| L2.behavior | testsuite + property + L0 differential | tolerance per `@py_compat`; ≥ 1000 fuzzed inputs per public fn | tomli ✅ |
| L2.perf | benchmark | ≥ 0.8× original on representative bench | M5+ — recorded, not gated |
| L3.pyo3 | PyO3-shaped wrapper | subprocess differential vs CPython | tomli ✅ |
| L3.dependents | top-5 dependents pass | downstream validation | M5+ |

Failure at any L2 gate routes diagnostic back to L1. Default
escalation threshold: 50 retries → human-readable failure report →
function marked `@py_compat(none)` with explanation.

## Provenance manifest

Written to `<crate_dir>/PROVENANCE.toml`. Schema pinned by `adr:0007` §3. Top-level sections:

```toml
[source]              # library, version, sha256 (full 64-hex), file_count
[oracle]              # runtime, runtime_version, oracle_module
[verification]        # seeds, fuzz_inputs_per_fn, divergences, known_failures
[router]              # strategy, models_used, ledger_entries
[build]               # toolchain, deterministic_id (blake3:<hex>), crate_layout_version
[gates]               # l0_spec_emitted, l1_files_emitted, l2_build, l2_behavior, l2_perf, l3_pyo3_wrapper, l3_downstream_dependents
```

`build.deterministic_id` is computed as:

```text
blake3(
    source_sha256_hex                   || b"\n" ||
    toolchain_string                    || b"\n" ||
    sorted_join(router_decision_ids, "\n")
)
```

Identical inputs ⇒ identical `deterministic_id`. This is the
constitution §2.4 promise made concrete.

## Synthetic provider — canned-response file format

```toml
schema_version = 1
oracle_runtime = "cpython 3.11"

[[entry]]
task = "translate"
function = "loads"
source_sha16 = "9a1adcc278853b5e"     # first 16 hex of source SHA-256
response_text = '''
... emitted Rust source ...
'''
```

The on-disk file is keyed by `(task, function)`. If the upstream
source SHA changes, every entry's `source_sha16` mismatches and the
synthetic provider returns
`LlmError::Provider { code: "synthetic-stale" }` — forcing the
curator to re-record. This is the staleness check that makes
synthetic mode safe (no silent translations).

## Invariants

- **No silent translations.** Every translated file carries a
  provenance header; every per-function emission carries a per-
  function provenance line.
- **Determinism.** Given identical
  `(source, toolchain, router decisions)`, the resulting crate is
  byte-identical. Verified by
  `tests/tomli_pipeline.rs::pipeline_deterministic_id_is_stable_across_runs`.
- **Closed loop.** Every gate failure feeds a diagnostic back into
  L1. No gate is bypassable.
- **Token cost is not a constraint.** Correctness, elegance,
  reproducibility are.
- **Differential fuzzing** is necessary, not sufficient — minimum
  1000 random inputs per public function; verified for tomli at
  `crates/cobrust-tomli/tests/tomli_fuzz.rs`.

## Non-goals

- Not a general-purpose Python-to-Rust transpiler. Targets Cobrust
  specifically; emitted code uses Cobrust's static core (or std-Rust
  in the M4 phase before the static core is consumed end-to-end).
- Not interactive — translation is a batch pipeline.

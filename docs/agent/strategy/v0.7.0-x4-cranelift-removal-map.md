## Constraint (do not violate)

`cobrust-jit` (REPL / JIT execution path) is **RETAINED** per ADR-0070 Â§X.3 Q2
(option a). Cranelift is therefore NOT fully removed from the workspace. The
removal target is the **AOT backend duplication**: `cranelift_backend` inside
`cobrust-codegen` (parallels `llvm_backend`). LLVM is now the sole AOT backend
(`default = ["llvm"]`).

**The shared substrate `crates/cobrust-codegen/src/lowering.rs` (+ `abi.rs`) imports
`cranelift-codegen` / `cranelift-frontend` and is a hard dependency of the retained
`cobrust-jit`.** Removing those two crates from cobrust-codegen breaks the JIT build.
Only `cranelift-module` + `cranelift-object` are cleanly droppable.

## Feature decision

With Cranelift gone, LLVM is the only backend â†’ the `llvm` feature stops being a
meaningful opt-in (`--no-default-features` would yield a codegen crate with no
working `emit()`). Two options:
- **(rec) Make `llvm` non-optional**: fold `inkwell` into hard `[dependencies]`,
  delete the `llvm` feature + all `#[cfg(feature="llvm")]` gates in codegen.
- Keep the feature name but document that disabling it produces a non-functional
  `emit()` (weaker; leaves a dead config).
Decide via ADR-0070 Â§X.4 sub-section. The `--no-default-features` escape hatch loss
is the secondary risk â€” audit CI jobs / downstream for that flag first.

## (d) Single biggest risk

Surgically excise `cranelift_backend.rs` + the `module`/`object` crates it alone uses
while leaving the Cranelift IR-construction crates (`cranelift-codegen`/`-frontend`)
that the retained JIT `lowering.rs` substrate depends on. A naive "Cranelift is gone,
drop all cranelift-*" breaks `lowering.rs` and the entire `cobrust-jit` build.

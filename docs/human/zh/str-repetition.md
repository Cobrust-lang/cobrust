# `str * int` 字符串重复 —— `"ab" * 3 == "ababab"`

> ADR-0097。`*` 运算符现在可以重复字符串:`s * n`(以及对称的
> `n * s`)会得到 `s` 重复 `n` 次的结果 —— 语义与 Python 完全一致。

## 先看示例

```cobrust
fn main() -> i64:
    print("ab" * 3)        # ababab
    print(3 * "ab")        # ababab   (对称 —— 两种顺序都可以)
    print("=" * 10)        # ==========
    let n: i64 = 2 + 1
    print("ab" * n)        # ababab   (计算得到的次数,不只是字面量)
    print(len("ab" * 3))   # 6        (结果是一个可用的 str)
    return 0
```

与 Python 一致:`"ab" * 3 == "ababab"`、`3 * "ab" == "ababab"`。

## 次数规则(CPython 3)

| 表达式 | 结果 | 原因 |
|---|---|---|
| `"x" * 0` | `""` | 次数为 0 得到空字符串 |
| `"x" * 1` | `"x"` | 次数为 1 是一份拷贝 |
| `"x" * -2` | `""` | **非正数**次数得到空字符串 —— 绝不 trap |
| `"ab" * 3` | `"ababab"` | 常见情形 |

零或负数次数在 Python 中不是错误,在 Cobrust 中同样不是 —— 程序会正常
构建、运行并以退出码 0 结束,打印空字符串。

## 按码点忠实(codepoint-faithful)

重复是把**整个**字符串拼接起来,因此边界绝不会落在某个多字节 UTF-8
码点的中间:

```cobrust
fn main() -> i64:
    print("é" * 2)         # éé      (U+00E9 被重复,绝不被切开)
    print(len("é" * 2))    # 4       (len 是字节长度:"éé" 是 4 字节)
    return 0
```

与 Python 一致:`"é" * 2 == "éé"`。

## 两种操作数顺序

Python 允许次数在任意一侧 —— `s * n` 与 `n * s` 含义相同。Cobrust 把
两者都归一化为“`str` 是被重复对象,`int` 是次数”:

```cobrust
let line: str = "-" * 40        # 右侧是 int
let line2: str = 40 * "-"       # 左侧是 int,结果相同
```

## 哪些仍然是类型错误

`*` 运算符仅在恰好一个操作数是 `str`、另一个是 `int` 时才表示重复。
其他情况都会在编译期被拒绝,与 Python 抛出 `TypeError` 完全一致:

- `"a" * "b"` —— `str * str` 是类型错误(不能用字符串作为次数)。
- `"a" * 1.5` —— `str * float` 是类型错误(次数必须是 `int`)。

## 设计说明

- **是新增,不是修复**:在 ADR-0097 之前,`"ab" * 3` 是一个干净的拒绝
  (`error[Type]: type mismatch: expected str, found i64`)—— 它并不是
  静默误编译。本次改动让这个常见惯用法**可用**,而不是修正一个错误值。
- **运行时**:`__cobrust_str_repeat(s, n)` 用一次预留容量的分配
  (`str::repeat`)构建结果,并铸造一块全新的 `str` 缓冲区(与
  `__cobrust_str_slice` 相同的铸造路径)。源 `s` 是借用(只读,不被
  消耗);全新的结果在其作用域恰好被释放一次。
- 这是 §2.5 “LLM 优先” 原则的直接体现:`"sep" * n` 是 LLM 经常写的
  惯用法(分隔线、填充、定宽对齐)。拒绝它会迫使 LLM 在一个非常常见的
  形式上去绕开自己的 Python 直觉。

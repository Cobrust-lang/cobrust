## 1. TL;DR

v0.2.0 closes **Phase F.3 in full**: five P0 language-completeness
features (`break`/`continue`, `for` loops, `list[str]`, `f64`, and
`dict[K,V]`) plus three P1 stdlib surfaces (`string`, `file IO`, and
the full `clone()` mitigation idiom) shipped in a single batch session
(6 sub-ADRs: 0050a–f), bringing the cumulative Phase F.3 P0 + P1
count to **8 items closed**. The language can now express daily string
processing, CSV slicing, file read/write, and dictionary lookup
programs with Python-shaped ergonomics and Rust-grade correctness
guarantees.

## 2. New Language Features

### `dict[K, V]` — insertion-order key/value store (M-F.3.4, ADR-0050d)

- Type parameters: `K ∈ {i64, str}` (Phase F.3); `f64` keys rejected
  at type-check (`TypeError::NotHashable`, NaN-breaks-Hash rationale).
- Backed by `indexmap::IndexMap<KeyEnum, ValueEnum>` — insertion order
  is a language guarantee from v0.2.0 forward.
- Operations shipped: `{}` empty literal, `{k: v, ...}` literal,
  `d[k]` read (panic+abort on missing key), `d[k] = v` write/insert,
  `key in d` membership (returns `bool`), `len(d)` (returns `i64`),
  `dict_is_empty(d)` (canonical `bool` predicate per §2.2).
- `Dict[f64, V]`, `if d:` (implicit truthiness), `{**other}` spread,
  and mutable-default `Dict` args are compile-rejected.
- Deferred to Phase G / next milestone: `for k, v in d.items():` iter
  desugar, `.get(k, default)`, `del d[k]`, dict comprehension,
  `d.keys()` / `d.values()` / `d.items()` / `d.copy()` method forms.

### `f64` first-class (M-F.3.3, ADR-0050c §f64 surfaces)

- IEEE-754 double precision: literals (`3.14`, `1e-3`), `inf`, `nan`.
- Explicit `as`-cast required: `(n as f64)`, `(3.9 as i64)`. No
  silent promotion.
- Math intrinsics: `sqrt`, `floor`, `ceil`, `round`, `abs`, `pow`,
  `sin`, `cos`, `tan`, `log`, `exp`.
- f-string format specs: `{x:.2f}` (fixed), `{x:e}` (scientific),
  `{x:g}` (general).

### `list[str]` with Str ownership (M-F.3.2, ADR-0050c)

- `list[str]` and `Ty::Str` are **non-Copy** uniformly (operand-level
  and drop-level). The compiler emits `__cobrust_str_drop` at scope
  exit and `__cobrust_str_clone` for explicit copies.
- `for s in xs:` over a `list[str]` clones each slot into the loop
  variable; the slot's ownership stays with `xs`.
- TD-1 (Str/List as phantom-Copy debt) is closed. Known honest-debt
  (Phase 2a walk-back): `list[T]` is Copy at the operand level, so
  passing a `list[str]` by value does not compile-reject the
  post-call use site; explicit-borrow syntax closes this in Phase G.

### `break` / `continue` (M-F.3.0, ADR-0050a)

- `break` exits the innermost loop; `continue` skips to the next
  iteration. Both are bare keywords — no label forms.
- Out-of-loop usage is a compile-time `TypeError`.
- Nested `for` and `while` loops: `break`/`continue` always bind the
  innermost enclosing loop.

### `for x in <list|range>:` loops (M-F.3.1, ADR-0050b)

- `range(start, stop)` materialises a `list[i64]` of `stop - start`
  slots (`start, start+1, …, stop-1`); empty range (start ≥ stop)
  skips the body.
- `for x in xs:` iterates any `list[T]` including `list[str]` and
  `list[i64]`.
- 3-argument `range(start, stop, step)`, string iteration
  (`for c in "hello":`), and the full iterator protocol are Phase G.

## 3. New Stdlib Surfaces

### string — 11 PRELUDE functions (M-F.3.5, ADR-0050e)

| Function | Signature | Notes |
|---|---|---|
| `split` | `(s: str, sep: str) -> list[str]` | `split("", ",") → [""]`; empty-sep Rust behavior |
| `join` | `(parts: list[str], sep: str) -> str` | `join([], sep) → ""`|
| `replace` | `(s: str, old: str, new: str) -> str` | empty-old inserts at every byte position |
| `trim` | `(s: str) -> str` | both-side whitespace |
| `find` | `(s: str, needle: str) -> i64` | `-1` if absent; `find(s, "") → 0` |
| `contains` | `(s: str, needle: str) -> bool` | `contains(s, "") → true` |
| `starts_with` | `(s: str, prefix: str) -> bool` | |
| `ends_with` | `(s: str, suffix: str) -> bool` | |
| `lower` | `(s: str) -> str` | |
| `upper` | `(s: str) -> str` | |
| `clone` | `(s: str) -> str` | deep-copy; LC-100 honest-debt mitigation idiom |

`find` returns an `i64` sentinel (`-1` for absent). The canonical
idiom is `if pos != -1:` — `if find(...):` is rejected as implicit
truthy/falsy per §2.2.

`clone(s)` is the mandated workaround for the LC-100 use-after-move
pattern: insert `clone()` before every non-final `str`-consuming PRELUDE
call so each surface call gets a fresh buffer. See
`docs/human/en/getting-started.md` §"Step 2.8" for the full idiom.

### file IO — 7 PRELUDE functions (M-F.3.6, ADR-0050f)

| Function | Signature | Returns | Notes |
|---|---|---|---|
| `read_file` | `(path: str) -> str` | file contents | Empty `str` on I/O error |
| `read_file_lines` | `(path: str) -> list[str]` | lines stripped of `\n`/`\r\n` | Trailing empty element preserved: `"a\nb\n" → ["a","b",""]` |
| `write_file` | `(path: str, contents: str) -> i64` | `0`=success, `1`=error | Creates or truncates |
| `append_file` | `(path: str, contents: str) -> i64` | `0`=success, `1`=error | Creates if absent |
| `stdin_read_all` | `() -> str` | stdin until EOF | Empty `str` on EOF |
| `stdout_write` | `(s: str) -> i64` | `0`/`1` sentinel | No trailing newline (differs from `print`) |
| `stderr_write` | `(s: str) -> i64` | `0`/`1` sentinel | stderr only; stdout unchanged |

i64-sentinel idiom:

```cobrust
let rc: i64 = write_file("/tmp/out.txt", "data")
if rc != 0:
    return rc
```

`print` always appends `\n` and always returns 0. `stdout_write`
omits the newline and surfaces the write result for pipe-closed
detection.

## 4. Known Limitations (Honest-Debt Enumeration)

These items are **documented known limitations**, not release blockers.
The verification run explicitly confirmed each claim below.

### LC-100 corpus regression — long-term deferral, no ETA

- **Root cause**: ADR-0050c §"Decision" Option A chose `Str = non-Copy`
  uniformly (operand-level and drop-level). Programs that pass the same
  `str` to multiple PRELUDE calls (e.g., `str_len(s)` followed by
  `str_at(s, i)`) trigger `UseAfterMove` at the second call.
- **Affected corpus**: ~100 LC-100 / leetcode programs that use the
  multi-read PRELUDE idiom. Pre-Wave-2 these compiled (Str was
  Copy-at-operand). Post-Wave-2 they fail.
- **Disposition (P10/user 2026-05-16)**: long-term tech debt. No Phase
  G closure target. Revisit when the language has external users + an
  explicit-borrow syntax (v0.3.0+ timeframe).
- **Workaround**: use the `clone()` idiom documented in
  `docs/human/en/getting-started.md` §"Step 2.8" — insert `clone(s)`
  before each non-final PRELUDE call on the same `str` binding.
- **Source**: `docs/agent/findings/lc100-str-use-after-move-regression-from-adr0050c.md`
  §"Long-term deferral — addendum 2026-05-16".

### 14 corpus-pattern-error e2e tests — documented pre-impl carry-forwards

The following test functions are deliberately authored as
corpus-pattern witnesses (either negative tests for behaviour that
requires Phase G, or pre-impl scaffolding whose `#[ignore]` markers
were not removed before the DEV sprint closed). They are **not bugs
in the implementation** — they document real gaps that map to specific
Phase G or later work items:

- `list_str_e2e.rs`: `f3ls22` (cross-block use-after-move — borrow
  check is intra-block at v0.2.0), `f3ls23`, `f3ls25`
- `string_stdlib_e2e.rs`: `f3str16`, `f3str17`, `f3str22`
- `file_io_e2e.rs`: `f3fio01`, `f3fio03`, `f3fio05`, `f3fio06`,
  `f3fio10`, `f3fio11`, `f3fio12`, `f3fio_bug03`

These inherit the same long-term deferral disposition as the LC-100
corpus. No scheduled closure sprint at v0.2.0.

### Dict sub-sprints e/f/g — deferred to Phase G (not bugs)

These are **planned next-milestone features**, not regressions:

- `for k, v in d.items():` iter desugar (requires MIR tuple-unpack).
- `.get(k, default)` method dispatch.
- `del d[k]` key deletion.
- Dict comprehension `{k: v for k, v in ...}`.
- `d.keys()` / `d.values()` / `d.items()` / `d.copy()` method forms.

### AI alpha surfaces remain `(alpha)`-labeled

`M-AI.0` (`llm_complete`), `M-AI.1` (`llm_dispatch`), and
`M-AI.2` (`llm_stream` / `llm_complete_structured` /
`llm_complete_with_tools`) are shipped as flat PRELUDE functions with
an `(alpha)` label. They return `""` (or `[]` for streaming) when
provider configuration is absent, rather than a descriptive runtime
error. Module-form sugar (`cobrust.llm.*`, `cobrust.prompt.*`) is
Phase F.4+.

## 5. Compiler Internals (for Contributors)

- **`indexmap = "2"` workspace dependency**: added for `dict` insertion-order backing. `IndexMap<KeyEnum, ValueEnum>` is the concrete type behind every `Dict[K, V]` value at runtime.

- **Symmetric Copy@operand / non-Copy@drop split for Str / List / Dict**:
  - `Ty::Str`: non-Copy uniformly (ADR-0050c Option A). Drop schedule emits `__cobrust_str_drop` at every reachable scope exit.
  - `Ty::List(_)`: Copy@operand (Phase 2a walk-back) + non-Copy@drop. Passing a `list[T]` by value does not consume the caller's binding; `__cobrust_list_drop_elems` fires at scope exit.
  - `Ty::Dict(_,_)`: Copy@operand (ADR-0050d Amendment 2026-05-16 dict walk-back, matching Phase 2a precedent) + non-Copy@drop. `__cobrust_dict_free` fires at scope exit.
  - The asymmetry between `Str` (fully non-Copy) and `List`/`Dict` (Copy@operand) is intentional and documented. Phase G closes the gap via explicit-borrow syntax.

- **F30 ADSD candidate — predicate-flip cascade discovery deficit**:
  The LC-100 regression is the strongest empirical case for the F30
  candidate. Two empirical baselines are now on record:
  - DG verify on `b364c3d` (post-Wave-1): 3127 passed / 0 failed (LC-100 green).
  - DG verify on `09006f6` (post-Wave-2): 3182 passed / 108 failed (LC-100 red; root cause = ADR-0050c Str=non-Copy predicate flip).
  The proposed SOP fix (shadow-flip dry-run with feature flag before
  §"Consequences" enumeration finalises) remains an open upstream
  proposal.

- **Dict codegen type-tag dispatch**: `__cobrust_dict_new(k_tag, v_tag, len)` reinterprets `k_size`/`v_size` as type tags (0=i64, 1=str). `__cobrust_dict_set_K_V` / `__cobrust_dict_get_K_V` / `__cobrust_dict_contains_K` dispatch on the static `(K, V)` shape. Legacy untyped `__cobrust_dict_*` symbols remain aliased to the `(i64, i64)` variants for M12.x backward compatibility.

## 6. Acknowledgments / Methodology

Phase F.3 closed in a single dispatch session (~24h wall time) with
6 P10-direct PAIR sprints and 3 audit teammates:

- **Wave 1** (P9-A + P9-B): `break`/`continue` (ADR-0050a) + `for` loop (ADR-0050b). Both features were substantially already shipped; Wave 1 closed the contract-seal, corpus baseline, and doc-coverage gap.
- **Wave 2** (P9-C + P9-D + P9-E): `f64` (ADR-0050c §f64 surfaces) + `list[str]` ownership (ADR-0050c full) + MIR drop schedule. LC-100 honest-debt surfaced and disposed at Wave 2 close.
- **Wave 3** (P9-F + P9-G + P9-H): `dict` codegen + `indexmap` (ADR-0050d) + string stdlib 11-fn bundle (ADR-0050e) + file IO 7-fn bundle (ADR-0050f).

ADSD discipline held throughout: 11+ findings filed across the 3
waves; 0 false-green claims reached external commits. The F30 ADSD
candidate (predicate-flip cascade discovery deficit) now has 2
empirical baselines as evidence.

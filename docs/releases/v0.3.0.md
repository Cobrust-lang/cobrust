## 2. §2.5 LLM-First Directions Shipped

### Direction A — Explicit `&s` borrow / let-rebind (ADR-0052a)

The single largest LLM-friendliness deficit at HEAD `v0.2.0` (LC-100
honest-debt empirical baseline) closes via three sub-ADRs:

- **ADR-0052a** (Wave 1) — `&expr` borrow surface: parser (`feat(parser,hir)`),
  HIR lowering, `Ty::Ref` + one-way call-site coercion at `synth_call_args`,
  MIR lowering `&expr` → `Operand::Copy`. §13 design lesson: bidirectional
  `Ref↔T` unify is the cascade root; one-way call-site coercion is the
  correct semantics. 3-attempt design recovery captured at `8f29189`.
- **ADR-0052f** (Wave 2 round 2 leanest) — Parser §8 cap relaxation for
  `&Call(Attr(...))` form (method-call borrow). Unblocks `&s.method()`
  parse path. Accepted at `94e5544`.
- **ADR-0052g** (Wave 2 round 2 follow-up) — Type-check support for
  `&CallResult`. Narrows `synth_expr` Borrow arm to admit Copy-primitive
  method returns (`i64` / `f64` / `bool`) while preserving
  `BorrowOfNonPlace` for non-Copy / non-method-form sites. Accepted at
  `15ef676`.

LLM-friendliness payoff: programs that read the same `str` binding
twice can write `str_len(&s)` then `find(&s, "x")` without
`use-after-move`. The `clone()` mitigation idiom (introduced at v0.2.0
for LC-100) remains supported but is no longer the canonical path.

### Direction B — Error UX rewrite (ADR-0052b)

Every error variant now carries the **fix path** as a structured field,
not as renderer-side prose. The §2.5 compile-time-catch rule converts
to a useful corrective signal only when the catch sites encode the
fix directly:

- **24 `TypeError::*` variants** in `crates/cobrust-types/src/error.rs`
  gain `suggestion: Option<&'static str>` populated at every
  `Err(TypeError::...)` construction site.
- **11 `MirError::*` variants** in `crates/cobrust-mir/src/error.rs`
  follow the same shape.
- **6 `LoweringError::*` variants** in `crates/cobrust-hir/src/error.rs`
  extend the pattern to HIR-lowering diagnostics (extended at `365181a`).
- **Total: 41 variants** all uniformly structured. ~55 construction
  sites updated to populate the field.
- The CLI renderer (`crates/cobrust-cli/src/error_ux.rs`) is now a thin
  structural pass-through; per-variant hint prose is no longer
  hard-coded in `match` arms.

Example pre/post:

```text
# pre v0.3.0
TypeError::ImplicitTruthiness at line 3: expected bool, found i64

# post v0.3.0
TypeError::ImplicitTruthiness at line 3: expected bool, found i64
  suggestion: change to `if x != 0:`
```

Downstream consumers — LSP `Diagnostic.relatedInformation`,
`--emit-json` agent loops, future tooling — read the suggestion field
directly. No round-trip through stringified renderer output.

### Direction C — `@py_compat` tier hard-bind to L2 (ADR-0052c)

ADR-0037 ("Reserved — py-compat hard-bind to L2 verifier") sat as a
3-year-deferred surface. v0.3.0 activates it. Marked
`superseded_by 0052c` at `ed357c0`.

- New `PyCompatTier` enum: `Strict | Semantic | Numerical { rtol } | None`.
  Parsed from `spec.toml` `py_compat: String` field at L0 spec
  extraction. Default = `Strict`.
- New `TierVerifier` replaces the legacy `AcceptAll` `BehaviorVerifier`.
  Per-tier behavior gate semantics:
  - `Strict` — bit-for-bit equality vs CPython oracle.
  - `Semantic` — equivalence up to documented Python-semantic delta.
  - `Numerical { rtol }` — element-wise rtol-bound numerical comparison.
  - `None` — gate auto-passes; logged as untrusted translation.
- New **tier-aware translation prompt** + **router strategy override**
  per tier — strict tier routes to consensus mode (highest quality);
  none tier routes to cost-strategy.
- Migration plan: `cobrust-tomli` + `cobrust-dateutil` + `cobrust-msgpack`
  re-tagged with explicit `py_compat` fields (deferred to follow-up
  sprint — translated crates still verify-green under TierVerifier
  default behavior).

### Direction D — Method-dispatch infrastructure (ADR-0052d-prereq)

Method-form (`s.split(",")`) is sugar over PRELUDE-fn (`split(s, ",")`)
— no vtable, statically resolved at type-check via `lower_call` rewrite
to PRELUDE-fn. The infrastructure ships in v0.3.0; full
method-call corpus migration (LC-100) deferred to **ADR-0052d-final**
in v0.3.1 timeframe.

**25 new method-form entries across 4 new dispatch tables:**

| Type | Methods | Count |
|---|---|---|
| Str | `len` / `split` / `replace` / `trim` / `find` / `contains` / `starts_with` / `ends_with` / `lower` / `upper` | 10 |
| List | `len` / `push` / `get` / `set` / `is_empty` | 5 |
| Float | `floor` / `ceil` / `abs` / `is_nan` / `is_finite` | 5 |
| Int | `abs` / `bit_count` / `pow` / `min` / `max` | 5 |
| **subtotal new** | | **25** |
| Dict (pre-existing) | `keys` / `values` / `items` / `get` / `copy` / `len` | 6 |

New `TypeError::UnknownMethod` variant for `s.unknown_method()` sites.
`&s.method()` parses as `&(s.method())` per ADR-0052 §F-G.3.

## 3. Constitutional Anchors (ADR-0051)

v0.3.0 codifies **CLAUDE.md §2.5** as the constitutional north star
binding every Phase G+ design trade-off. The amendment was accepted at
`708981b` (ADR-0051) and now governs:

- **Compile-time-catch-errors rule** — every `TypeError::*` /
  `MirError::*` variant is a "successful catch". Direction B converts
  the catch to a useful corrective signal (fix path encoded structurally).
- **Maximize-overlap-with-training-data rule** — prefer syntax matching
  the Python+Rust training-corpus distribution. Direction D method-form
  sugar is the first ratified application of this rule.
- **Four binding priority directions** A/B/C/D — all shipped in v0.3.0.

The audit-teammate rubric ("did this design respect §2.5's
compile-time-catch + training-data-overlap rules?") is enforced on
every Phase G+ design ADR.

## 4. Infrastructure: CLI Tempdir RAII

A long-standing operational hazard closes in v0.3.0:

- **Root cause** (`a5710ff`): `cobrust build` / `cobrust run` created
  `/tmp/cobrust-build-*` intermediate-`.o` dirs without RAII drop.
  Leaked an estimated 235G of temp data on the DG workstation over 6
  months of CI iteration.
- **Fix** (`cd6f69b`): wrap intermediate-artifact paths in
  `tempfile::TempDir` so Rust's `Drop` semantics guarantee cleanup
  even on panic / cancellation / signal. `tempfile` promoted from
  dev-dep to runtime dep.
- **Verification** (`3adc98b`): DG attestation — PRE-build tempdir
  count = POST-build tempdir count.
- **Test** (`4444629`): integration test asserts `build` and `run`
  cleanup invariants.

Closes the operational debt; bound forever via the integration test.

## 6. Known Limitations (Honest-Debt Enumeration)

These items are **documented known limitations**, not release blockers.

### TEST-author-pattern errors (~30 across Phase G)

Per the Phase F.3 honest-debt precedent (v0.2.0 §4 "14 corpus-pattern-
error e2e tests"), Phase G also surfaces test functions deliberately
authored as corpus-pattern witnesses. The Wave 1 + Wave 2 round
ADR-0052a closure documented **12 0052a-prefix residuals** as
TEST-author-pattern-errors at `9c89222`. Wave 2 round 2 added further
pattern witnesses for the `&Call(Attr(...))` form.

These are not bugs in the implementation — they document real gaps
that map to specific Phase G+ work items. **0 non-Phase-G regressions
vs main HEAD baseline** confirmed at every sub-ADR ratification point.

### LC-100 corpus method-form migration — deferred to v0.3.1

ADR-0052d-final (full method-call corpus migration replacing PRELUDE-fn
call-sites with method-form sugar) is **not in Phase G scope**. The
method-dispatch infrastructure ships in v0.3.0 (`a4e599e` ratification
amendment); the corpus rewrite scopes into v0.3.1.

Programs that opt into method-form today work correctly; the LC-100
corpus continues to use PRELUDE-fn + `clone()` per v0.2.0 idiom until
the migration sprint lands.

### Dict sub-sprints e/f/g — still Phase H+ scope

The Phase F.3 v0.2.0 deferral list (dict `.items()` iter desugar,
`.get(k, default)`, `del d[k]`, comprehension, `.keys()` / `.values()`
method forms) remains deferred. Method-dispatch infrastructure
(Direction D) is the **substrate** for these — they will land as
follow-up dict-method completions in v0.3.x or Phase H.

### AI alpha surfaces remain `(alpha)`-labeled

`M-AI.0` / `M-AI.1` / `M-AI.2` PRELUDE functions ship with the `(alpha)`
label inherited from v0.2.0. ADR-0049 alpha-honesty hardening lanes
remain active. Module-form sugar (`cobrust.llm.*`) is post-Phase-H.

## 7. Compiler Internals (for Contributors)

- **`Ty::Ref` introduced**. `Ty::Ref(Box<Ty>)` represents `&T` references
  at the type level; constructed by the `&expr` borrow surface. Borrow-
  check intra-block discipline unchanged; cross-block borrow-check is
  Phase H.
- **One-way call-site coercion**. `synth_call_args` coerces `&T → T` at
  call sites where the parameter is `T` and the argument is `&T`. No
  reverse coercion. This is the ratified semantics post-3-attempt
  design recovery (ADR-0052a §13).
- **`suggestion: Option<&'static str>` field uniform across 3 error
  enums**. Zero-allocation rendering preserved (static literals only).
  Dynamic format-arg suggestions (e.g. `format!("try {name}.is_some()")`)
  remain out of scope per ADR-0052b §11.
- **`TierVerifier` replaces `AcceptAll`**. L2 verification gate now
  dispatches per-tier behavior contracts. Backward-compatible:
  unstamped translations default to `Strict`.
- **Method-dispatch via `try_synth_<type>_method`**. Pattern
  generalized from `try_synth_dict_method` (check.rs:1008). Each method
  arm rewrites at `lower_call` to the corresponding PRELUDE-fn call —
  no runtime overhead vs PRELUDE-fn form.
- **`TypeError::UnknownMethod`** — new variant for `value.unknown_method()`
  catch sites. Suggestion field populated with the closest-name hint.

## 10. Post-Phase-G Roadmap (ADR-0054)

The user-listed order (LLVM Backend / LSP / Debugger / REPL JIT) is
**NOT** the ship order. §2.5 ROI rerank inverts to:

| Rank | Phase | Surface | §2.5 ROI | Est. wall-time |
|---|---|---|---|---|
| 1 | **J** | LSP server | highest | 2-3w agent-velocity |
| 2 | **H** | Self-host type checker | high | 3w agent-velocity |
| 3 | **I** | REPL JIT (M14.1) | medium | 1w agent-velocity |
| 4 | **K** | LLVM Backend | §2.5-neutral | 3-4w agent-velocity |
| 5 | **L** | Debugger | §2.5 ~0 | 1w agent-velocity |

LSP first because IDE-LLM session quality is the largest agent
multiplier (Cursor / Continue / Cody / Aider all consume LSP
`Diagnostic.relatedInformation` + `CodeAction.title` — Direction B's
structured `suggestion` is the LSP-feed substrate). Debugger last
because agents read stderr + retry rather than single-step.

Total runway ~10-12 weeks post-G at agent-velocity (4-8x compression
vs human estimate per Wave 1+2 empirical calibration).

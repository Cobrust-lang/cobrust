# Cobrust v0.1.2 — Prebuilt Binary Version-String Fix

> Patch release on top of `v0.1.1` (commit `769a5d8`).
> No language / runtime / translator behavior changes — release-build fix only.

## Why v0.1.2 (not retag v0.1.1)

v0.1.1's prebuilt binary tarballs (`cobrust-v0.1.1-*.tar.gz`) were compiled
from the v0.1.1 tag commit (`769a5d8`), which still had workspace version
`0.1.0-beta` in `Cargo.toml`. Cargo embeds the package version into the
binary at compile time, so running `cobrust --version` on the v0.1.1
prebuilt binary returned `cobrust 0.1.0-beta` — inconsistent with the
release page label `v0.1.1`.

The v0.1.2 patch:

- Bumps workspace `version` to `0.1.2` BEFORE the tag (correct order).
- Re-builds tier-1 prebuilt binaries with the bumped version embedded.
- Establishes a permanent "bump-before-tag" SOP for all future releases.

v0.1.1 remains tagged and downloadable as a historical artifact; a
deprecation banner on the v0.1.1 release page points users at v0.1.2 for
a clean install.

## What's fixed

- Prebuilt binary `cobrust --version` now reports the matching tag version.
- `cargo install --git ...` continues to compile cleanly (same path as v0.1.1).
- All release-notes curl commands now use `curl -fsSL` (was `-L`) so a 404
  surfaces as a clear error instead of silently downloading an HTML page.

## Install path (same as v0.1.1, with `-fsSL`)

**Option A — cargo install (requires Rust toolchain):**
```bash
cargo install --git https://github.com/Cobrust-lang/cobrust cobrust-cli
```
(crates.io publish queued for v0.2.0)

**Option B — prebuilt binary (no Rust required):**
```bash
# macOS arm64
curl -fsSL https://github.com/Cobrust-lang/cobrust/releases/download/v0.1.2/cobrust-v0.1.2-aarch64-apple-darwin.tar.gz | tar xz
sudo mv cobrust /usr/local/bin/

# Linux x86_64
curl -fsSL https://github.com/Cobrust-lang/cobrust/releases/download/v0.1.2/cobrust-v0.1.2-x86_64-unknown-linux-gnu.tar.gz | tar xz
sudo mv cobrust /usr/local/bin/

# Linux arm64
curl -fsSL https://github.com/Cobrust-lang/cobrust/releases/download/v0.1.2/cobrust-v0.1.2-aarch64-unknown-linux-gnu.tar.gz | tar xz
sudo mv cobrust /usr/local/bin/
```

> **Platform support note**: v0.1.2 ships prebuilt binaries for macOS arm64
> (Apple Silicon) and Linux x86_64/arm64. Apple Intel (`x86_64-apple-darwin`)
> and Windows (`x86_64-pc-windows-msvc`) builds are queued for v0.2.0.
